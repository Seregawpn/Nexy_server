# CRM Instruction Source of Truth Analysis

## Метаданные
- Ассистент: codex
- Тип: analysis
- Дата: 2026-01-05

## Запрос/цель
Проанализировать текущие практики (чек-листы/инструкции) и предложить единые правила формирования инструкций для CRM/канбана.

## Контекст
- Файлы: Docs/IMPLEMENTATION_CHECKLIST.md, Docs/DUPLICATE_CONFLICT_RACE_PREVENTION_CHECKLIST.md, Docs/MANUAL_TESTING_CHECKLIST.md, Docs/PHASE1_TESTING_CHECKLIST.md
- Документы: Docs/PROJECT_REQUIREMENTS.md, Docs/ARCHITECTURE_OVERVIEW.md, Docs/ASSISTANT_COORDINATION_PROTOCOL.md, Docs/CODEX_PROMPT.md, Docs/assistant_exchange/TEMPLATE.md
- Ограничения: один источник истины, минимизация дублирования

## Решения/выводы
- В проекте уже есть множество чек-листов, но нет единого реестра и единой инструкции по их созданию.
- Источник истины для инструкций должен быть в Docs и опираться на существующий шаблон.
- Нужны единые правила структуры: цель, триггер, шаги, ожидаемые артефакты, критерии DoD.

## Открытые вопросы
- Нужно ли объединять часть чек-листов или достаточно реестра и правил?
- Какие чек-листы будут считаться обязательными для CRM прогресса?

## Следующие шаги
- Зафиксировать правила единого формата инструкций.
- Создать реестр инструкций/чек-листов с владельцами и назначением.
- Определить, какие из них дают факты для прогресса CRM.
