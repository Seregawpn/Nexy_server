# Super Kanban 2.0 Roadmap Review

## Метаданные
- Ассистент: codex
- Тип: analysis
- Дата: 2026-01-05

## Запрос/цель
Оценить план Super Kanban 2.0 и проверить соответствие архитектурным принципам (единый источник истины, отсутствие дублирования, минимизация сложности).

## Контекст
- Файлы: Docs/PROJECT_REQUIREMENTS.md, Docs/ARCHITECTURE_OVERVIEW.md, Docs/ASSISTANT_COORDINATION_PROTOCOL.md, Docs/ANTIGRAVITY_PROMPT.md, Docs/CODEX_PROMPT.md
- Документы: AGENTS.md, Docs/assistant_exchange/TEMPLATE.md
- Ограничения: один источник истины для данных, без локальных вторичных хранилищ

## Решения/выводы
- План реалистичен, если закрепить единый источник данных (JSON) и централизовать бизнес-логику (dependencies, releases, epics) в одном модуле.
- Визуализации (Gantt, sidebar, lock icons) должны быть чистым рендерингом без дублирования состояния.
- Release Workflow допустим как операция, переводящая карточки из канбана в архив releases.

## Открытые вопросы
- Нужна ли строгая блокировка перехода в Done или достаточно warning?
- Требуется ли поддержка нескольких релизных веток одновременно?

## Следующие шаги
- Подтвердить правила блокировок и поведение Release Workflow.
- Сформировать минимальный список полей V2 и миграцию данных из текущей схемы.
