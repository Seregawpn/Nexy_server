# Client Browser Install Plan

## Метаданные
- Ассистент: codex
- Тип: analysis
- Дата: 2026-01-18
- ID (INS-###): INS-005

## Diagnosis
Запрос на установку браузера на клиенте конфликтует с текущей архитектурой: браузерная автоматизация и установка сейчас принадлежат серверному модулю `browser_use`.

## Root Cause
Архитектура закрепляет `browser_use` за сервером → установка Chromium выполняется сервером → клиент не имеет ответственного модуля для установки.

## Optimal Fix
Перенести `browser_use` (или хотя бы установку Chromium) на клиента через модуль/интеграцию и оставить единый владелец решения на клиенте.

## Verification
Проверить, что установка запускается на клиенте и сервер не пытается устанавливать Chromium при старте.

## Запрос/цель
Определить, как сделать установку Chromium на клиентской стороне.

## Контекст
- Файлы: server/modules/browser_use/module.py, client/Docs/ARCHITECTURE_OVERVIEW.md, client/Docs/PRODUCT_CONCEPT.md
- Документы: Docs/ARCHITECTURE_OVERVIEW.md, Docs/PROJECT_REQUIREMENTS.md
- Ограничения: не ломать текущую архитектуру без явного запроса

## Решения/выводы
- Если `browser_use` остается на сервере, установка Chromium должна оставаться на сервере.
- Чтобы установка происходила у клиента, нужно переместить владение `browser_use` на клиента.

## Открытые вопросы
- Нужен ли полный перенос `browser_use` в client или только отключение серверной установки?

## Следующие шаги
- Уточнить, должен ли браузер исполняться на клиенте.
- При подтверждении — подготовить план переноса модуля и обновления конфигурации.
