# Browser Install And TTS

## Метаданные
- Ассистент: codex
- Тип: analysis
- Дата: 2026-01-18
- ID (INS-###): INS-005

## Diagnosis
Сервер при старте инициирует установку Chromium и локально озвучивает прогресс через `say`, что расходится с ожиданием клиентского TTS.

## Root Cause
Логика browser_use размещена на сервере и использует Playwright авто-установку + macOS `say`, обходя клиентский аудио-канал.

## Optimal Fix
Цель: оставить серверную автоматизацию браузера, но убрать локальную озвучку и централизовать аудио на клиенте.

## Verification
DoD: сервер не воспроизводит `say`, клиент получает прогресс через стандартный канал/логи.

## Запрос/цель
Объяснить источник серверного запроса на установку браузера и неочевидной речи при наличии TTS.

## Контекст
- Файлы: server/server/modules/browser_use/module.py, server/server/config/unified_config.yaml, client/modules/browser_progress/core/types.py
- Документы: Docs/ARCHITECTURE_OVERVIEW.md, Docs/PROJECT_REQUIREMENTS.md
- Ограничения: без реархитектуры, без дублирования логики

## Решения/выводы
- Установка браузера выполняется на сервере модулем browser_use (Feature F-2025-015).
- Озвучивание идет через системный `say`, а не через TTS пайплайн.

## Открытые вопросы
- Нужно ли отключать browser_use целиком на сервере в текущем окружении?

## Следующие шаги
- Согласовать отключение `say` и/или перенос уведомлений в клиентский TTS.
