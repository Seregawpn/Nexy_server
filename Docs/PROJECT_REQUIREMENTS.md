# Project Requirements (Root Index)

Root-документ является индексом.

Канонические требования:
- Клиент: `client/Docs/PROJECT_REQUIREMENTS.md`

Связанные каноны:
- Клиентская архитектура: `client/Docs/ARCHITECTURE_OVERVIEW.md`
- Клиентский flow контракт: `client/Docs/FLOW_INTERACTION_SPEC.md`
- Серверная архитектура: `server/server/Docs/ARCHITECTURE_OVERVIEW.md`

Правило централизации:
- Требования runtime не дублируются в root.
- Изменения требований вносятся в канон владельца и проверяются профильными verify-скриптами.
